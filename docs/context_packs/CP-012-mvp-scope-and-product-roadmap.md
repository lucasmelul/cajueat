# Objetivo

Definir exactamente qué vamos a construir primero.

Este documento existe para evitar uno de los mayores riesgos del proyecto:

Construir demasiado.

Todo lo que no sea indispensable para validar el producto deberá esperar.

---

# Resumen Ejecutivo

El MVP de CajuEat no busca competir con Google Maps.

Busca demostrar que una IA especializada puede ayudar a tomar mejores decisiones gastronómicas.

Si logramos eso, el resto llegará naturalmente.

---

# Objetivo del MVP

Que un usuario pueda:

- abrir la aplicación;
- descubrir un lugar interesante;
- conversar con el Brain;
- decidir rápidamente;
- visitar el lugar;
- dejar feedback;
- mejorar el Brain.

Ese ciclo completo valida la hipótesis principal.

---

# La Gran Hipótesis

Si una persona recibe mejores recomendaciones que con Google Maps o Instagram, volverá a utilizar CajuEat.

Todo el MVP gira alrededor de validar esa idea.

# Criterios de Éxito

No mediremos únicamente DAUs.

Queremos medir.

- calidad de recomendaciones;
- satisfacción;
- velocidad para decidir;
- porcentaje de recomendaciones aceptadas;
- cantidad de conocimiento nuevo;
- crecimiento del Brain.

---

# Filosofía

Antes de agregar una funcionalidad debemos responder.

¿Hace que el usuario tome una mejor decisión?

Si no.

No entra.

---

# Producto Mínimo

Queremos el producto más pequeño capaz de enamorar usuarios.

No el producto con más funcionalidades.
