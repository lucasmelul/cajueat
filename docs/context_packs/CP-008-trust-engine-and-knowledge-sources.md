# Objetivo

Definir cómo el Brain construye confianza.

El objetivo no es tener más información.

El objetivo es saber en qué información confiar.

La confianza será uno de los activos más importantes de CajuEat.

---

# Resumen Ejecutivo

Google muestra información.

Instagram muestra contenido.

TikTok muestra tendencias.

Los influencers muestran recomendaciones.

Los usuarios dejan reviews.

El Brain combina todas esas señales y genera un criterio propio.

No copia ninguna fuente.

Aprende de todas.

---

# Filosofía

No queremos construir:

> La base de restaurantes más grande.

Queremos construir:

> El sistema que mejor decide en quién confiar.

Ese será nuestro diferencial.

---

# Principios

## Ninguna fuente tiene la verdad

Toda fuente puede equivocarse.

---

## Toda afirmación tiene evidencia

No existe conocimiento sin respaldo.

---

## Toda evidencia tiene un peso

No todas las señales valen lo mismo.

---

## La confianza cambia

Un restaurante puede mejorar.

Puede empeorar.

Una fuente puede ganar o perder credibilidad.

Todo evoluciona.

# Curadores

Un curador es una entidad del Brain.

No solamente una cuenta de Instagram.

Cada curador posee:

- identidad;
- especialidad;
- cobertura;
- reputación;
- evolución;
- coincidencia con usuarios.

---

# Especialización

No existe un mejor influencer.

Existe un mejor influencer para cada tema.

Ejemplos.

Uno puede ser excelente en:

- café.

Otro en:

- parrillas.

Otro en:

- sushi.

El Brain aprende eso.

---

# Curated Lists

Las listas poseen muchísimo valor.

Ejemplos.

Las mejores hamburguesas.

Los mejores brunch.

Las mejores cafeterías.

El Brain no guarda únicamente los restaurantes.

Guarda también:

- autor;
- fecha;
- orden;
- contexto;
- argumentos.

---

# Consenso

Cuando varias fuentes coinciden.

La confianza aumenta.

Ejemplo.

Un restaurante recomendado por:

- Buenos Paladaires.
- Salt Argentina.
- Comunidad Caju.

Tiene una señal muy fuerte.

---

# Desacuerdo

El desacuerdo también genera conocimiento.

Ejemplo.

Google:

4.8

Usuarios recientes:

"La calidad cayó."

Influencers:

"No volvería."

El Brain detecta una tendencia.

---

# Freshness

La información reciente vale más.

No porque la antigua sea incorrecta.

Sino porque el mundo cambia.

---

# Evidencia

Toda afirmación importante puede responder:

¿Por qué?

Ejemplo.

"Excelente para una cita."

↓

Porque:

- 42 experiencias recientes.
- 5 curadores.
- 91% coincidencia con usuarios similares.

---

# Transparencia

Cuando el usuario quiera.

Puede conocer el razonamiento.

No mostramos algoritmos.

Mostramos evidencia.

---

# Personalización

El Brain aprende con quién coincide cada usuario.

Ejemplo.

Lucas coincide un 93% con Buenos Paladaires.

Entonces las recomendaciones de ese curador tendrán mayor peso para Lucas.

Otro usuario puede coincidir con otra persona.

---

# Importación de Conocimiento

Queremos que alimentar el Brain sea extremadamente fácil.

Ejemplos.

"Analizá esta cuenta."

"Importá este Reel."

"Leé esta lista."

"Este es el nuevo menú."

El Brain hace el resto.

# Objetivos

- Construir confianza.
- Detectar cambios.
- Encontrar joyas ocultas.
- Descubrir tendencias antes que otros.
- Priorizar calidad sobre popularidad.

---

# Casos de Uso

## Caso 1

Un restaurante nuevo todavía tiene pocas reviews.

Pero cinco curadores excelentes ya lo recomiendan.

El Brain lo detecta antes que Google.

---

## Caso 2

Un restaurante muy famoso bajó muchísimo.

El Brain detecta la caída antes que las plataformas tradicionales.

---

## Caso 3

El usuario coincide mucho con un influencer.

Las recomendaciones se personalizan automáticamente.

---

# Reglas

- Nunca copiar rankings.
- Nunca depender de una sola fuente.
- Toda confianza puede cambiar.
- Toda afirmación importante debe tener evidencia.
- El Brain construye su propio criterio.

---

# Decisiones Tomadas

✅ Los influencers son entidades del Brain.

✅ Existen distintos pesos según dominio.

✅ La confianza es dinámica.

✅ El Brain calcula consenso.

✅ La evidencia es obligatoria.

---

No mostrar estrellas como elemento principal.

No construir rankings estáticos.
