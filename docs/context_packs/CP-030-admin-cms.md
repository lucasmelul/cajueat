# Objetivo

Definir el Backoffice del producto.

No queremos un panel administrativo tradicional.

Queremos un CMS conversacional.

---

# Filosofía

Administrar CajuEat debería sentirse como conversar con el Brain.

---

# Ejemplos

"Importá este Reel."

"Agregá este restaurante."

"Corregí este horario."

"Analizá esta lista."

"Mostrame restaurantes sin menú."

---

# Funciones

- importar conocimiento;
- editar entidades;
- revisar confianza;
- aprobar cambios;
- analizar fuentes;
- detectar inconsistencias.

---

# Paneles

- entidades;
- fuentes;
- usuarios;
- conocimiento;
- operaciones.

---

# Objetivos

Reducir drásticamente el tiempo de administración.

---

# Qué NO hacer

No construir formularios enormes.
