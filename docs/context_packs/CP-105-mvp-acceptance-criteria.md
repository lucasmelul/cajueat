# CP-105 — MVP Acceptance Criteria

**Estado:** Draft

---

# Objetivo

Definir cuándo podremos decir que CajuEat MVP está terminado.

---

# El MVP está listo cuando un usuario puede:

✅ Abrir la aplicación.

✅ Descubrir restaurantes desde el mapa.

✅ Conversar naturalmente con el Brain.

✅ Obtener recomendaciones personalizadas.

✅ Comparar alternativas.

✅ Abrir la ficha inteligente de un restaurante.

✅ Guardar restaurantes.

✅ Crear colecciones.

✅ Recibir explicaciones del Brain.

✅ Compartir conocimiento mediante conversación.

✅ Compartir un Reel o un link para alimentar el Brain.

✅ Dejar feedback luego de una visita.

✅ Obtener Caju Points.

✅ Mejorar futuras recomendaciones gracias a ese feedback.

---

# El Brain está listo cuando puede:

✅ Recordar usuarios.

✅ Aprender continuamente.

✅ Explicar sus decisiones.

✅ Manejar incertidumbre.

✅ Integrar múltiples fuentes.

---

# El producto NO está listo si:

❌ Parece Google Maps.

❌ Parece TripAdvisor.

❌ Parece un chatbot genérico.

❌ Requiere muchos formularios.

❌ No sorprende al usuario en los primeros minutos.

---

# Estado

Checklist oficial para el lanzamiento del MVP.
