# CP-018 — Discovery Engine

**Estado:** Draft
**Versión:** 1.0
**Owner:** Product
**Consumidores:** Claude Code, Claude Design, Product

---

# Objetivo

Definir cómo el Brain descubre lugares para el usuario.

El descubrimiento constituye la principal razón para abrir CajuEat.

No queremos que el usuario piense:

> "Voy a buscar un restaurante."

Queremos que piense:

> "Quiero descubrir algo increíble."

---

# Resumen Ejecutivo

El Discovery Engine es el sistema responsable de encontrar oportunidades gastronómicas relevantes.

No busca restaurantes.

Busca la mejor decisión posible para el usuario en ese momento.

---

# Filosofía

Google responde búsquedas.

CajuEat genera descubrimientos.

---

# Principios

## Relevancia antes que Popularidad

Un restaurante poco conocido pero perfecto para el contexto debe aparecer antes que uno famoso.

---

## Contexto antes que Distancia

La cercanía importa.

Pero no siempre es el factor principal.

---

## Calidad antes que Cantidad

Cinco recomendaciones excelentes valen más que cien mediocres.

---

## Descubrimiento antes que Exploración

El usuario no debería tener que navegar veinte minutos.

El Brain hace la selección.

---

# Variables Analizadas

Antes de recomendar, el Brain considera:

- ubicación;
- horario;
- clima;
- presupuesto;
- compañía;
- gustos;
- historial;
- restaurantes guardados;
- experiencias pasadas;
- tendencias;
- disponibilidad de información;
- confianza.

---

# Tipos de Descubrimiento

## Espontáneo

El usuario abre la app.

El Brain ya tiene recomendaciones.

---

## Conversacional

El usuario expresa una necesidad.

---

## Geográfico

El usuario explora una zona.

---

## Inspiracional

El usuario no sabe qué quiere.

El Brain propone.

---

## Contextual

Una situación modifica las recomendaciones.

Ejemplo.

Está lloviendo.

---

# Descubrimiento Diario

Cada vez que el usuario abre la aplicación debería sentir que hay algo nuevo.

No necesariamente restaurantes nuevos.

Puede ser:

- un lugar olvidado;
- una nueva recomendación;
- un restaurante que mejoró;
- un menú especial;
- una nueva experiencia.

---

# Joyas Ocultas

Uno de los objetivos estratégicos.

Descubrir lugares increíbles antes que el resto.

No depender de popularidad.

---

# Diversidad

El Brain evita recomendar siempre lo mismo.

Debe equilibrar.

- confianza;
- exploración;
- novedad.

---

# Serendipia

El usuario debería sorprenderse.

Ejemplo.

"Jamás hubiera encontrado este lugar solo."

---

# Colecciones Dinámicas

El Discovery Engine también genera colecciones.

Ejemplos.

Las mejores cafeterías para trabajar hoy.

Lugares ideales para un día lluvioso.

Brunch cerca tuyo.

No son listas estáticas.

Se generan dinámicamente.

---

# Descubrimiento Cercano

Cuando el usuario camina.

El Brain puede detectar oportunidades cercanas.

Sin ser invasivo.

---

# Descubrimiento Histórico

El Brain también recuerda.

"Hace mucho que no vas a..."

---

# Descubrimiento Social

En versiones futuras.

Podrá descubrir lugares donde personas con gustos similares tuvieron grandes experiencias.

No por amistad.

Por afinidad gastronómica.

---

# Objetivos

- reducir tiempo para decidir;
- aumentar descubrimiento;
- sorprender;
- construir confianza;
- incentivar exploración.

---

# Reglas

- Nunca mostrar demasiadas opciones.
- Nunca recomendar únicamente por popularidad.
- Toda recomendación debe tener contexto.
- El descubrimiento debe sentirse natural.

---

# Decisiones Tomadas

✅ El Brain selecciona.

✅ El usuario explora.

✅ Pocas recomendaciones.

✅ Muchísima relevancia.

---

# Decisiones Abiertas

- Frecuencia de actualización.
- Recomendaciones predictivas.
- Descubrimiento offline.

---

# Qué NO hacer

No construir un buscador.

No ordenar por distancia.

No ordenar por estrellas.

No mostrar rankings infinitos.
