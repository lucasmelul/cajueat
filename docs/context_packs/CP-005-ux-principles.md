# CP-005 — User Experience (UX) Principles

**Estado:** Draft
**Versión:** 1.0
**Owner:** Product
**Consumidores:** Claude Design, Claude Code, Product

---

# Objetivo

Definir cómo debe sentirse CajuEat.

Este documento NO define pantallas.

Define la experiencia.

Toda decisión de diseño debe respetar estos principios.

---

# Resumen Ejecutivo

La mayor diferencia de CajuEat no será la IA.

Será cómo se siente usarla.

Queremos que el usuario piense:

> "Nunca tuve que pensar demasiado."

Todo debe sentirse natural.

---

# Filosofía

El usuario nunca debería sentir que está usando una aplicación gastronómica.

Debería sentir que está hablando con alguien que conoce perfectamente la ciudad y también lo conoce a él.

---

# Principios Fundamentales

## Invisible First

La interfaz desaparece.

La protagonista es la decisión.

No queremos botones llamativos.

Queremos decisiones simples.

---

## Conversation First

Siempre que una tarea pueda resolverse hablando, debe poder hacerse.

Ejemplos:

"No sé qué comer."

"Busco una cafetería para trabajar."

"Quiero sorprender a mi pareja."

---

## Decision First

No mostramos información por mostrar.

Todo debe ayudar a decidir.

Ejemplo.

No mostrar:

★★★★☆ 4.6

Mostrar:

Ideal para una primera cita.

---

## Context First

Toda recomendación depende del contexto.

El mismo restaurante puede ser excelente o una mala elección dependiendo de:

- clima;
- horario;
- compañía;
- presupuesto;
- objetivo;
- tiempo disponible.

---

## Explainability

El usuario nunca debería preguntarse:

"¿Por qué me recomendó esto?"

El Brain siempre debe poder responder.

---

## Progressive Disclosure

No mostrar toda la información de entrada.

Mostrar solamente lo necesario.

Profundizar únicamente cuando el usuario lo pida.

---

## Calm Interface

No queremos una aplicación ruidosa.

Sin banners.

Sin popups.

Sin promociones.

Sin elementos que compitan entre sí.

---

## Fast Decisions

Toda interacción debe ahorrar tiempo.

No agregar pasos innecesarios.

---

## Delight

Debe haber pequeños momentos memorables.

Ejemplos.

Cómo aparece una recomendación.

Cómo se mueve el mapa.

Cómo responde el Brain.

Pequeñas animaciones.

Nunca excesivas.

---

# Sensaciones buscadas

El usuario debería sentir:

- confianza;
- curiosidad;
- calma;
- rapidez;
- inteligencia;
- personalización.

Nunca:

- ansiedad;
- confusión;
- sobrecarga;
- ruido.

---

# Navegación

La navegación debe sentirse invisible.

No queremos cinco tabs permanentes.

No queremos menús complejos.

El usuario debería llegar a cualquier funcionalidad mediante:

- conversación;
- mapa;
- acciones contextuales.

---

# Descubrimiento

El descubrimiento debe ser natural.

Nunca mostrar cientos de restaurantes.

El Brain selecciona.

El usuario explora.

---

# Búsqueda

Buscar no debería sentirse como completar un formulario.

Ejemplos.

"Sushi."

"Algo romántico."

"Quiero merendar."

"Con terraza."

El Brain interpreta.

---

# Personalización

La personalización nunca debe sentirse invasiva.

Debe sentirse útil.

Ejemplo.

"No te recomiendo este porque sé que buscabas algo tranquilo."

---

# Feedback

El feedback nunca debe sentirse como una encuesta.

Debe sentirse como una conversación.

---

# Aprendizaje

El usuario nunca debería sentir que está entrenando una IA.

Simplemente comparte experiencias.

El Brain hace el resto.

---

# Errores

Cuando el Brain no sabe.

Debe decirlo.

Cuando duda.

Debe explicarlo.

Cuando necesita información.

Debe preguntarla.

Nunca inventar.

---

# Performance Percibida

La sensación de velocidad es tan importante como la velocidad real.

El usuario nunca debería sentir que espera.

---

# Consistencia

Todo el producto debe hablar el mismo idioma.

Las mismas acciones.

Los mismos conceptos.

Los mismos patrones.

---

# Acciones Principales

Toda acción importante debe responder una de estas categorías.

- Descubrir.
- Preguntar.
- Guardar.
- Compartir.
- Aprender.
- Planificar.

Si una funcionalidad no entra en alguna de estas categorías, probablemente no pertenezca al MVP.

---

# Acciones que NO queremos

- Completar formularios largos.
- Leer textos enormes.
- Navegar menús complejos.
- Configurar decenas de opciones.
- Aprender a usar la aplicación.

---

# Inspiraciones

No copiar.

Inspirarse.

Nivel de calidad buscado.

- Apple Maps.
- Arc Browser.
- Linear.
- Raycast.
- Perplexity.
- Airbnb.
- Spotify.
- Notion.

---

# Regla de Oro

Antes de agregar una funcionalidad debemos preguntarnos:

> ¿Hace que decidir sea más fácil?

Si la respuesta es no.

No pertenece a CajuEat.

---

# Decisiones Tomadas

✅ Conversación como interfaz principal.

✅ Mapa como Home.

✅ Personalización progresiva.

✅ Explicabilidad obligatoria.

✅ Diseño minimalista.

✅ Sin gamificación invasiva.

---

# Decisiones Abiertas

- Navegación definitiva.
- Sistema de gestos.
- Voice Mode.
- Modo oscuro.
- Accesibilidad avanzada.

---

# Qué NO hacer

No copiar Google Maps.

No copiar Instagram.

No copiar TripAdvisor.

No convertir CajuEat en una app llena de botones.

No diseñar pensando en features.

Diseñar pensando en decisiones.
