# Roadmap

Este roadmap es de producto, no técnico. Las fases describen qué se define y valida, no cómo se construye.

## Fase 1 — Blueprint

- Consolidar documentación de producto (este set de docs).
- Escribir y aprobar los PRDs iniciales (ver [prds/](prds)).
- Diseño de experiencia a partir de [design-brief.md](design-brief.md).
- Definir arquitectura técnica a partir de [codex-brief.md](codex-brief.md).

## Fase 2 — Vertical Slice y Beta privada

- Construir el alcance definido en [mvp.md](mvp.md) para una sola ciudad.
- Cargar una base inicial de conocimiento (restaurantes, fuentes curadas — ver [sources-and-curators.md](sources-and-curators.md)).
- Beta cerrada con un grupo reducido de usuarios reales.
- Medir si el Brain empieza a aprender de forma visible (recomendaciones que mejoran con el uso).

## Fase 3 — Lanzamiento e iteración

- Lanzamiento abierto en la ciudad inicial.
- Iterar sobre Trust Engine, gamificación y calidad conversacional con datos reales de uso.
- Evaluar expansión: nuevas ciudades, nuevas fuentes, features postergadas del MVP (ver [mvp.md](mvp.md)).

## Qué decide cuándo pasar de fase

El criterio para avanzar de fase no es una fecha fija, sino:

- **Fase 1 → 2**: los PRDs del MVP están aprobados y no hay ambigüedad de producto pendiente en [product-decisions.md](product-decisions.md) que bloquee el desarrollo.
- **Fase 2 → 3**: la beta privada demuestra que el Trust Engine produce recomendaciones confiables y que el flujo de captura de conocimiento efectivamente alimenta al Brain.

## Fuera de este roadmap

Decisiones de timing específicas (fechas, duración de la beta, criterios numéricos de graduación) no están definidas todavía. Ver [product-decisions.md](product-decisions.md).
