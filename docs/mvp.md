# MVP

## Criterio

El MVP de CajuEat debe ser **pequeño pero diferencial**. No es una versión reducida de una app de restaurantes genérica: debe hacer sentir, desde el primer uso, la tesis de producto ("CajuEat ayuda a decidir") aunque el alcance sea acotado.

Cada feature del MVP existe porque:

1. es necesaria para que el Brain empiece a aprender, o
2. es necesaria para que el usuario sienta la diferencia frente a una app tradicional.

Si una feature no cumple ninguna de las dos, se posterga.

## Incluye

- **PWA** como único cliente.
- **Living Map** como Home ([PRD-001](prds/PRD-001-home-living-map.md)).
- **Prompt Bar** conversacional y **Brain Card**.
- **Chat básico con el Brain** ([PRD-002](prds/PRD-002-conversational-brain.md)).
- **Ficha de restaurante** ([PRD-003](prds/PRD-003-restaurant-experience.md)).
- **Guardar lugares**.
- **Captura de conocimiento** (voz, foto, link) ([PRD-004](prds/PRD-004-knowledge-capture.md)).
- **Feedback conversacional post-visita** ([PRD-005](prds/PRD-005-post-visit-feedback.md)).
- **Caju Points**, versión básica ([PRD-006](prds/PRD-006-caju-points.md)).
- **Trust Engine**, versión simple ([PRD-008](prds/PRD-008-trust-engine.md)).
- **Fuentes y curadores**, versión simple, como señal de confianza ([PRD-007](prds/PRD-007-sources-and-curators.md)).
- **Onboarding mínimo** ([PRD-009](prds/PRD-009-onboarding.md)).
- **Perfil de usuario** básico ([PRD-010](prds/PRD-010-user-profile.md)).

## No incluye (postergado)

- Reservas.
- Delivery.
- Red social (seguir usuarios, feeds sociales, comentarios públicos).
- Planificación compleja de múltiples salidas/itinerarios.
- Funcionalidad de amigos / grupos.
- Ranking avanzado o leaderboards competitivos.
- Múltiples ciudades (el MVP se lanza en una sola ciudad).
- API pública.

Ver [roadmap.md](roadmap.md) para cuándo se retoma cada uno de estos ítems.

## Relación con los principios

El MVP no sacrifica los cinco principios rectores (ver [vision.md](vision.md)) por velocidad. Sacrifica **alcance** (menos ciudades, menos features, menos integraciones), no **calidad de experiencia** ni **confianza**. Un MVP más chico pero fiel a Brain First / Trust First es preferible a uno más grande que se sienta genérico.
